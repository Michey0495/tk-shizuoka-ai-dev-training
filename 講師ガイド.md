## 2. 各セッションの講師向けメモ

### Session01: オリエンテーション

**強調ポイント**

- 本研修は「デモを見て学ぶ」形式であり、手を動かす時間はないことを明示
- 2時間で「生成AI開発の現在地」を俯瞰できることを伝える
- エンジニア32名という規模なので、積極的に質問を促す雰囲気作りを

**リスク・ガバナンスの説明ポイント**

- 生成AIは便利だが、機密情報の扱いには注意が必要
- 企業での導入時には、情報漏洩リスク・著作権リスクを必ず検討する
- 「とりあえず使ってみる」から「ルールを決めて使う」へ

**よくある質問への回答例**

Q: 資料は後でもらえますか？
A: 本日の資料URLは閉会時にお伝えします。配布用フォルダにまとめてあります。

Q: 自社で導入する場合の費用感は？
A: GitHub Copilot Businessは1ユーザー月額$19程度。Cursor Proは月額$20。詳細はSession03で触れます。

### Session02: VSCodeでの生成AI活用ベストプラクティス

**強調ポイント**

- GitHub Copilotは「補完」だけではなく「対話」もできる時代になった
- Agentモードの登場で、複数ファイルにまたがる修正も一度の指示で可能に
- ショートカットを覚えるだけで生産性が大きく変わる

**デモの見せ方**

1. まず完成形を見せる（ゴールを明確に）
2. 次にプロンプトを見せる（何を指示したか）
3. 生成過程をリアルタイムで見せる（AIの動きを実感）
4. 修正点があれば、追加プロンプトで対応する過程も見せる

**Vibe Coding デモの流れ**

1. 空のフォルダから開始
2. 「ブラウザで動くシンプルなAIチャットアプリを作って」と指示
3. HTML/CSS/JSが生成される過程を見せる
4. ブラウザでプレビュー
5. 「デザインを改善して」で追加指示

**よくある質問への回答例**

Q: GitHub Copilotの精度は実際どうですか？
A: 定型的なコードは高精度。ただしビジネスロジック特有の部分は人間のレビューが必須。補完率70-80%くらいのイメージで、残りは人間が調整する。

Q: Copilot ChatとAgentモードの違いは？
A: Chatは対話で質問に答えてくれる。Agentモードはコードベース全体を理解して、実際にファイルを編集してくれる。Agentは「実行」まで踏み込む。

Q: セキュリティは大丈夫？コードが外部に送られる？
A: GitHub Copilot Businessプランでは、コードがモデル学習に使われないことが保証されている。ただし、プロンプトとして送信はされるので、機密情報を直接書かないルール作りは必要。

### Session03: AI駆動開発における選択肢

**強調ポイント**

- ツールは日進月歩。「今日のベスト」が来月も同じとは限らない
- 大事なのは「Rulesを育てる」というマインド
- どのツールを使っても、ルール（.cursorrules, CLAUDE.md等）は共通して活用できる

**IDE比較の要点**

| IDE | 特徴 | 向いているチーム |
|-----|------|-----------------|
| Cursor | 高機能・Tab補完が優秀 | 新規プロジェクト、スタートアップ |
| Antigravity | UIがシンプル・軽量 | 既存プロジェクトの改修 |
| Kiro | 仕様駆動に強み | ウォーターフォール寄りの現場 |
| Windsurf | Flowが独自 | 探索的な開発 |

**CLI比較の要点**

| CLI | 特徴 | 向いているユースケース |
|-----|------|----------------------|
| Claude Code | 深い推論・長いコンテキスト | 複雑なリファクタリング |
| Codex | OpenAI製・GPT系 | 既存OpenAIユーザー |
| Gemini CLI | Google製・無料枠あり | 試験的な利用 |

**MCPの説明ポイント**

- Model Context Protocol: LLMが外部データにアクセスするための標準仕様
- Google Drive、Notion、GitHub等と連携可能
- 「AIに社内ドキュメントを読ませる」が実現できる
- ただしセキュリティ設定は慎重に

**よくある質問への回答例**

Q: 結局どのツールがおすすめですか？
A: チームの状況による。まずGitHub Copilotを導入して効果を実感し、そこから必要に応じてCursor等を検討するのが現実的。

Q: Cursorは無料で使えますか？
A: 無料プランあり（月500回の補完制限）。本格利用にはProプラン（月$20）が必要。

Q: MCPを使うとセキュリティリスクが上がりませんか？
A: 上がる。だからこそ、どのデータソースに接続するか、どのユーザーに許可するかを事前に設計する必要がある。野良MCPサーバーには接続しない。

### Session04: 質疑応答・ディスカッション

**Q1: AIの進化による技術の均一化と成長機会の損失について**

ファシリテーションのポイント:
- 「AIがコードを書いてくれるなら、エンジニアの存在意義は？」という不安に寄り添う
- 一方で「AIを使いこなせるエンジニア」の価値が上がっている現実も伝える
- 成長機会は「消える」のではなく「変わる」という視点を提示

落としどころ:
- 基礎力（アルゴリズム、設計思想）は引き続き重要
- AIは「手段」であり、何を作るかの「目的」を決めるのは人間
- 若手は「AI込みで」スキルを積む時代になった

**Q2: ジュニア/ミドル層エンジニアの育成について**

ファシリテーションのポイント:
- シニアエンジニアの視点と、若手の視点の両方を引き出す
- 「AI時代に必要なスキル」は何かを参加者に考えてもらう

落としどころ:
- コードレビューの仕方が変わる（AIが書いたコードをどうレビューするか）
- 「なぜこう書いたか」を説明できる力が求められる
- ペアプロの相手がAIになる場面も増える

## 4. トラブルシューティング

### GitHub Copilotが動かない

**症状**: 補完候補が表示されない

**対処手順**:
1. VSCode右下のCopilotアイコンを確認（有効か？）
2. アイコンをクリックして再認証を試す
3. VSCodeを再起動
4. それでもダメならバックアップ動画に切り替え

### インターネット接続が不安定

**症状**: Copilotの応答が遅い・途切れる

**対処手順**:
1. 会場のWi-Fiから有線に切り替え（可能なら）
2. モバイルテザリングをバックアップとして準備
3. オフラインで説明できるスライド部分を先に進める
4. 回復したらデモに戻る

### Agentモードが使えない

**症状**: Agentモードのオプションが表示されない

**対処手順**:
1. VSCode Insiders版でないと使えない場合がある（バージョン確認）
2. GitHub Copilot Chatの設定でAgent機能を有効化
3. 使えない場合は通常のCopilot Chatでデモを進行

### 画面共有がうまくいかない

**症状**: プロジェクターに映らない、解像度がおかしい

**対処手順**:
1. ディスプレイ設定でミラーリングを確認
2. 解像度を1920x1080に固定
3. バックアップPC（あれば）に切り替え
4. 最悪の場合、ホワイトボードで概念説明

### 参加者からの想定外の質問

**対処方針**:
- 即答できない質問は「良い質問ですね。後ほど調べてお伝えします」で保留可
- 他の参加者に「どう思いますか？」と振るのも有効
- 研修後のフォローアップメールで回答する旨を伝える

## 5. 質疑応答の想定Q&A

### 技術的な質問

**Q: GitHub Copilotの学習データに自社コードが使われることはありますか？**

A: GitHub Copilot Businessプラン以上では、ユーザーのコードがモデルの再学習に使用されないことがGitHubから明言されています。ただし、推論時にはプロンプトとしてコードが送信されるため、機密情報を直接コードに書き込まないポリシーは必要です。

**Q: AIが生成したコードの著作権はどうなりますか？**

A: 法的にはまだグレーゾーンが多い状況です。現時点では、AIが生成したコードは「ツールを使って人間が作成したもの」として扱うのが一般的です。ただし、他者のコードと酷似するものが生成される可能性はあるため、重要な部分はライセンス確認を推奨します。

**Q: AIにコードを書かせると、エンジニアのスキルが落ちませんか？**

A: 電卓の登場で暗算力は落ちましたが、数学全体の能力は落ちていません。同様に、ボイラープレートを書く力は落ちるかもしれませんが、設計力・問題定義力・AIへの指示力といった上位スキルの重要性が増しています。

**Q: CursorとGitHub Copilotはどちらが良いですか？**

A: ユースケースによります。既存のVSCode環境を活かしたいならGitHub Copilot、より積極的なAI支援を求めるならCursor。両方試して比較するのがおすすめです。企業導入の観点では、GitHub Copilot BusinessはEnterpriseサポートが充実しています。

**Q: Claude CodeとCursor、どちらを使うべきですか？**

A: 用途が異なります。Cursorは「IDE内で完結する開発」向け。Claude Codeは「CLI操作やスクリプト実行を含む広範な作業」向け。両方を使い分けるエンジニアも増えています。

### 導入・運用に関する質問

**Q: 自社で導入する場合、何から始めればいいですか？**

A: 以下のステップを推奨します:
1. まず個人で無料版を試す（1-2週間）
2. チーム内の有志でパイロット導入（1ヶ月）
3. 効果測定とセキュリティポリシーの策定
4. 正式導入とルール整備

**Q: セキュリティポリシーはどう作ればいいですか？**

A: 最低限、以下を定めることを推奨します:
- AIに送信してよい情報の範囲（社内情報のランク分け）
- 利用可能なAIサービスのホワイトリスト
- 生成コードのレビュープロセス
- インシデント発生時の報告フロー

**Q: ROI（費用対効果）はどう測定していますか？**

A: 定量的には「コード記述時間の削減率」「バグ修正時間の削減率」が測りやすいです。GitHubの調査では、Copilot利用者の55%が「タスク完了が速くなった」と回答しています。定性的には開発者満足度も指標になります。

### 組織・キャリアに関する質問

**Q: AI時代にエンジニアとして生き残るには何を学ぶべきですか？**

A: 以下の3つを重視することをお勧めします:
1. プロンプトエンジニアリング（AIへの指示力）
2. アーキテクチャ設計（全体を俯瞰する力）
3. ドメイン知識（業務理解）

コードを「書く」力よりも、何を作るべきかを「定義する」力の価値が高まっています。

**Q: ジュニアエンジニアはAIを使うべきですか、使わないべきですか？**

A: 使うべきです。ただし「なぜそのコードになったか」を必ず理解する習慣をつけてください。AIが出力したコードをそのまま使うのではなく、1行ずつ読んで意図を把握する。その繰り返しが学習になります。

**Q: コードレビューの仕方は変わりますか？**

A: 変わります。レビューのポイントが「書き方」から「意図」にシフトします。「なぜこの設計にしたのか」「AIの提案をなぜ採用/却下したのか」を確認するようになります。
