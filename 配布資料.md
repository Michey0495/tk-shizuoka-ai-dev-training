## 1. 研修概要とゴール

### 本研修の目的

2024年後半から2025年にかけて、AI支援開発ツールは「補助」から「主導」へと役割を変えつつある。本研修では、最新のAI駆動開発手法とツール群を俯瞰し、自身の開発ワークフローに取り入れるための判断軸を獲得することを目指す。

### 到達目標

1. Vibe Coding、仕様駆動開発、AIエージェントなど最新概念を説明できる
2. AI搭載IDE・CLI搭載LLMの特徴と使い分けを理解する
3. 外部コンテキスト引用（MCP等）の仕組みと活用シーンを把握する
4. 自社・自チームへの導入検討に必要な観点を持ち帰る

## 2. 用語集

### Vibe Coding（バイブコーディング）

Andrej Karpathy氏が2025年2月に提唱した開発スタイル。コードの詳細を逐一確認せず、AIの出力を「雰囲気」で受け入れながら開発を進める手法。プロトタイピングや個人開発で威力を発揮する一方、本番コードへの適用には慎重な判断が必要。

### 仕様駆動開発（Spec-Driven Development）

従来の「コードを書いてからテスト」ではなく、「仕様・要件を自然言語で記述し、AIがコードを生成」するアプローチ。PRD（製品要件定義書）やユーザーストーリーをAIへのインプットとして活用する。

### AIエージェント（AI Agent）

単発の質問応答ではなく、複数のステップを自律的に実行するAIシステム。ファイル操作、コマンド実行、Web検索などを組み合わせてタスクを完遂する。Cursor Agent Mode、Claude Code、Codex CLI等がこの形態を採用。

### MCP（Model Context Protocol）

Anthropic社が2024年11月に公開したオープンプロトコル。AIモデルが外部データソース（ファイルシステム、データベース、API等）へ標準化された方法でアクセスする仕組みを提供する。

### コンテキストウィンドウ

LLMが一度に処理できるトークン数の上限。Claude 3.5は200Kトークン、GPT-4 Turboは128Kトークン。大規模コードベースを扱う際の制約となる。

### RAG（Retrieval-Augmented Generation）

検索拡張生成。外部知識ベースから関連情報を取得し、LLMの回答生成に活用する手法。社内ドキュメントや最新情報を反映した回答が可能になる。

### Rules / System Prompt

AIツールの振る舞いを制御する事前指示。プロジェクト固有のコーディング規約、使用ライブラリ、禁止事項などを定義する。Cursorの`.cursorrules`、Claude Codeの`CLAUDE.md`が該当。

## 3. ツール比較表

### AI搭載IDE

| 項目 | Cursor | Windsurf | Kiro |
|------|--------|----------|------|
| ベース | VS Code Fork | VS Code Fork | VS Code Fork |
| 開発元 | Anysphere | Codeium | AWS |
| 主要モデル | Claude, GPT-4, 独自 | 独自モデル中心 | Claude (Bedrock) |
| Agentモード | あり（強力） | あり（Cascade） | あり（Spec駆動） |
| 価格 | $20/月〜 | $15/月〜 | 無料（プレビュー） |
| 特徴 | 市場リーダー、エコシステム充実 | 高速補完、独自UI | 仕様駆動開発に特化 |
| Rules機能 | .cursorrules | .windsurfrules | spec/ フォルダ |

### CLI搭載LLM

| 項目 | Claude Code | Codex CLI | Gemini CLI |
|------|-------------|-----------|------------|
| 開発元 | Anthropic | OpenAI | Google |
| 主要モデル | Claude Sonnet/Opus | GPT-4o, o3 | Gemini 2.5 Pro |
| 実行環境 | ターミナル | ターミナル | ターミナル |
| 価格 | API従量課金 | API従量課金 | 無料枠あり |
| 特徴 | 長文コンテキスト、並列実行 | シンプル設計 | Google連携 |
| サンドボックス | あり | あり | あり |

### 選定の観点

- チーム規模: 大規模ならCursor/Windsurf、個人ならCLI系も有力
- 既存環境: VS Code資産を活かすならFork系IDE
- セキュリティ要件: オンプレLLMが必要ならContinue + Ollama
- 予算: 無料枠重視ならKiro、Gemini CLI

## 5. デモで使用したプロンプト例

### Session02: VSCode + GitHub Copilot

#### Agentモードでのタスク指示

```
このプロジェクトにReactでチャットUIを追加してください。
要件:
- メッセージ入力フォーム
- 送信ボタン
- メッセージ履歴の表示
- ローカルストレージへの保存
```

#### 並列駆動開発（Git Worktree）

```
現在のブランチから新しいworktreeを作成し、
feature/auth ブランチで認証機能を、
feature/dashboard ブランチでダッシュボードを
並行開発できる状態にしてください。
```

### Session03: AI駆動開発ツール

#### Cursorでのリファクタリング指示

```
@codebase このプロジェクトの認証ロジックを確認し、
以下の改善を行ってください:
1. 重複コードの統合
2. エラーハンドリングの強化
3. TypeScript型定義の追加
```

#### Claude CodeでのMCP活用

```
MCPでGoogle Driveに接続し、
「設計書」フォルダ内のドキュメントを参照して、
その仕様に基づいたAPIエンドポイントを実装してください。
```

#### Rulesファイルの例（.cursorrules）

```markdown
# プロジェクトルール

## 技術スタック
- TypeScript 5.x
- React 18
- Tailwind CSS

## コーディング規約
- 関数コンポーネントのみ使用
- hooksは use プレフィックス
- any型は禁止

## 禁止事項
- console.log を本番コードに残さない
- 直接DOM操作しない
```

## 7. 後日の学習ステップ

### Week 1: 環境構築と基本操作

- [ ] 使用するツールを1つ選定（Cursor推奨）
- [ ] 無料トライアルまたはアカウント作成
- [ ] 既存の小規模プロジェクトで補完機能を試す
- [ ] Rulesファイルを作成し、プロジェクト固有の指示を設定

### Week 2: Agentモードの活用

- [ ] Agentモードでファイル作成・編集を実行
- [ ] コードレビュー依頼のプロンプトを試す
- [ ] テストコード自動生成を体験
- [ ] 失敗パターン（意図しない変更）への対処を学ぶ

### Week 3: チーム導入の検討

- [ ] セキュリティポリシーとの整合性を確認
- [ ] 社内ガイドライン案を作成
- [ ] 小規模チームでのパイロット運用を計画
- [ ] 効果測定の指標を定義（コード量、レビュー時間等）

### Week 4: 発展的な活用

- [ ] MCP連携で社内データソースを接続
- [ ] CLI系ツール（Claude Code等）を試す
- [ ] 複数ツールの使い分けパターンを確立
- [ ] ナレッジ共有（社内勉強会、ドキュメント化）

### 継続的な学習

- 公式ブログ・リリースノートを定期購読
- 新機能は小規模タスクで検証してから本番適用
- チーム内でのプロンプト共有・改善サイクルを回す
